Paradigma Evolução

Programação Linear


É uma improtante área de otimização por várias razões. Muitos Problemas práticos em pesquisa operacional podem ser expressos como problemas de programação linear. Historicamente, ideias de programação linear inspiram muitos dos conceitos de teoria de otimização, tais como dualidade e decomposição.

Linguagem de Programação de Baixo Nível


Assembly


Inicialmente não foi otimizado nenhum tradutor para compilar ou mostrar as linguagens da primeira geração. As instruções de programação eram inseridas através dos interruptores localizados no painel frontal de sistemas de computadores. O principal benefício das linguagens de baixo nível é que o código escrito pelo usuário é muito rápido e eficiente, desde que este código seja diretamente executado pela CPU.

Programação Modular


Onde o desenvolvimento das teorias de programação é feito através de módulos, que são interligados entre sí através de uma interface comum.

As Promeiras Linguagens


Fortran, ALGOL


É representada pela linguagem assembly, que é uma notação legivel por humanos para o código de máquina, que uma arquitetura de computador específica usa. A linguagem de máquina, que é um mero padrão bin, torna-se legível pela substituição dos valores em bruto por simbolos chamados mineumponicos, ou seja, enquanto o computador sabe por que a instrução-máquina 1A-21 (1011000001100001) faz, para os programadores é mais fácil recordar a representação equivalente em instruções mineumónicas.

Programação Estruturada


Com esta programação é possível que todos os programas possam ser reduzidos a apenas três estruturas: sequência, decisão e repetição.

As procedurais e estruturadas


PASCAL, C

São as primeiras linguagens consideradas de alto nível. É uma linguagem de programação de fácil entendimento para o ser humano, incluindo coisas como variáveis com nomes.

Programação Orientada a Objetos


É uma programação de análise, projeto e programação de sistemas de software baseado na composição e interação entre diversas unidades de softwares chamados de objetos.

Linguagens que geram programas em outras linguagens


Java, C++, Linguagens de consulta (SQL)


A principal diferença entre essas linguagens, é que as primeiras linguagens são procedurais que descrevem como fazer algo, enquanto a outra descreve o que se quer que seja feito.
